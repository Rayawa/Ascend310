### 2. 平板专属UI布局设计

**设计理念：**
- 左右分栏布局：左侧显示图片和检测结果，右侧显示详细信息和控制面板
- 更大的Canvas尺寸：从360px增加到600px
- 更大的字体和间距
- 卡片式设计，充分利用大屏空间

**布局结构：**
```
┌─────────────────────────────────────────────────────────┐
│                    Header (渐变色)                        │
│  X-Ray Detection | YOLOv5s + Ascend 310B | 状态指示器    │
└─────────────────────────────────────────────────────────┘
┌──────────────────────┬──────────────────────────────────┐
│                      │                                  │
│   图片显示区域        │    控制面板                      │
│   (Canvas 600px)     │    - 选择图片按钮                │
│                      │    - 引擎状态                    │
│                      │    - 操作提示                    │
│                      ├──────────────────────────────────┤
│                      │    类别说明                      │
│                      │    - 5个类别卡片                 │
│                      │    - 实时统计                    │
├──────────────────────┴──────────────────────────────────┤
│                      检测结果                            │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐                   │
│  │ 1  │ │ 2  │ │ 3  │ │ 4  │ │ 5  │                   │
│  └────┘ └────┘ └────┘ └────┘ └────┘                   │
└─────────────────────────────────────────────────────────┘
```

**响应式设计策略：**
- 使用`display.getDefaultDisplaySync()`获取屏幕信息
- 根据屏幕宽度判断是否为平板（> 600dp）
- 动态设置Canvas尺寸和布局结构

**具体实现：**

1. **设备类型判断**：
   ```typescript
   import { display } from '@kit.ArkUI';
   
   @State isTablet: boolean = false;
   @State canvasSize: number = 360;
   
   aboutToAppear() {
     let displayInfo = display.getDefaultDisplaySync();
     let screenWidth = px2vp(displayInfo.width);
     this.isTablet = screenWidth > 600;
     this.canvasSize = this.isTablet ? 600 : 360;
     
     // 直接初始化引擎
     this.initEngine();
   }
   ```

2. **平板专属布局**：
   - 使用`if (this.isTablet)`条件渲染
   - 实现左右分栏布局（Row + Column）
   - 左侧：图片显示区域（Canvas）
   - 右侧：控制面板 + 类别说明 + 检测结果

3. **视觉优化**：
   - 更大的字体：标题28px → 32px，正文14px → 16px
   - 更大的间距：padding和margin增加20-30%
   - 更大的按钮：高度50px → 60px
   - 更大的卡片圆角：12px → 16px

## Risks And Compatibility

### 模型部署风险

1. **best.om格式不匹配**：
   - 风险：模型输出格式与代码假设不一致
   - 缓解：打印输出tensor的shape信息，必要时修改后处理
   - 验证：执行推理并检查结果

2. **内存不足**：
   - 风险：模型过大导致内存分配失败
   - 缓解：使用ACL_MEM_MALLOC_HUGE_FIRST策略
   - 验证：监控内存使用情况

### UI适配风险

1. **设备类型判断不准确**：
   - 风险：某些设备屏幕尺寸在临界值附近
   - 缓解：使用更可靠的设备类型API（如果可用）
   - 回退：提供手动切换布局的选项

2. **性能问题**：
   - 风险：复杂布局导致渲染性能下降
   - 缓解：使用LazyForEach优化列表渲染
   - 验证：测试滚动流畅度

3. **兼容性问题**：
   - 风险：不同平板设备显示效果不一致
   - 缓解：使用相对尺寸和百分比布局
   - 验证：在多种设备上测试

## Additional Notes

### 平板UI设计参考

参考HarmonyOS官方设计规范：
- 最小触摸目标：44dp
- 内容边距：24dp（平板）/ 16dp（手机）
- 字体缩放比例：平板比手机大20-30%
- 卡片圆角：16dp（平板）/ 12dp（手机）
- 阴影深度：平板可使用更深的阴影

### 性能优化建议

1. **图片加载优化**：
   - 使用缩略图预览
   - 大图按需加载
   - 考虑使用WebP格式

2. **渲染优化**：
   - 使用`LazyForEach`优化列表
   - 避免频繁的状态更新
   - 合理使用`@Watch`监听

3. **内存优化**：
   - 及时释放PixelMap
   - 控制Canvas绘制频率
   - 使用对象池复用对象
